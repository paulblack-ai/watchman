Create `tests/test_gate2.py` with pytest + pytest-asyncio tests:

**Fixtures:**
- `tmp_db`: Creates temp SQLite DB with `init_db()`, yields path, cleans up
- `sample_enriched_card`: Creates a SignalCard with review_state='approved', enrichment_state='complete', enrichment_data set to a valid IcebreakerToolEntry JSON. Inserts into DB and returns card_id.
- `sample_entry`: Returns an IcebreakerToolEntry fixture for testing

**Unit Tests (pytest.mark.unit):**

1. **test_build_gate2_card_with_re_enrich**: Build Gate 2 blocks with `can_re_enrich=True`. Assert blocks contain approve_gate2, reject_gate2, and re_enrich action IDs.

2. **test_build_gate2_card_without_re_enrich**: Build Gate 2 blocks with `can_re_enrich=False`. Assert blocks contain approve_gate2 and reject_gate2 but NOT re_enrich.

3. **test_build_gate2_card_shows_enrichment_details**: Build Gate 2 blocks. Assert the entry name, description, and at least one capability appear in the block text.

4. **test_build_gate2_confirmed_approved**: Build confirmed blocks with "gate2_approved". Assert text contains "Approved".

5. **test_build_gate2_confirmed_rejected**: Build confirmed blocks with "gate2_rejected". Assert text contains "Rejected".

6. **test_build_gate2_confirmed_re_enriching**: Build confirmed blocks with "re_enriching". Assert text contains "re-enrichment" or similar.

**Integration Tests (pytest.mark.integration):**

7. **test_gate2_approve_updates_state**: Create enriched card in DB. Simulate Gate 2 approval by calling `repo.set_gate2_state(card_id, 'gate2_approved')`. Reload card and assert gate2_state == 'gate2_approved' and gate2_reviewed_at is not None.

8. **test_gate2_reject_updates_state**: Create enriched card in DB. Call `repo.set_gate2_state(card_id, 'gate2_rejected')`. Assert gate2_state == 'gate2_rejected'.

9. **test_enrichment_attempt_increment**: Create enriched card in DB. Call `repo.increment_enrichment_attempt(card_id)` twice. Assert enrichment_attempt_count == 3.

10. **test_re_enrich_capped_at_3_attempts**: Create card with enrichment_attempt_count=3. Assert that `card.enrichment_attempt_count >= 3` means can_re_enrich should be False (test the logic, not the action handler directly).

11. **test_find_enriched_pending_gate2**: Create two cards: one enriched+pending gate2, one enriched+gate2_approved. Call `repo.find_enriched_pending_gate2()`. Assert only the pending one is returned.

Use the same DB test patterns from `tests/test_enrichment.py` (init_db, insert card, run queries).
