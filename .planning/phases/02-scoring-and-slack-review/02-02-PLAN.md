  <action>
    1. Create `src/watchman/slack/__init__.py` (empty).

    2. Create `src/watchman/slack/app.py`:
       - `create_slack_app() -> App`: creates Bolt `App(token=os.environ["SLACK_BOT_TOKEN"])`, calls `register_actions(app)` and `register_commands(app)`, returns app
       - `start_socket_mode(app: App) -> threading.Thread`: creates `SocketModeHandler(app, os.environ["SLACK_APP_TOKEN"])`, runs `handler.start()` in a daemon thread, returns the thread
       - Use logging for startup messages

    3. Create `src/watchman/slack/blocks.py` with Block Kit builder functions:
       - `build_signal_card_blocks(card: SignalCard, score: RubricScore) -> list[dict]`:
         - Section block with card title as mrkdwn link, score line formatted as "8.2 -- strong taxonomy fit" per user decision (use `top_dimension` human-readable label + `composite_score`)
         - Context block with source name, tier badge, date
         - Actions block with 4 buttons: Approve (primary style), Reject (danger style), Snooze 30d, Details — each with `action_id` and card `id` as `value`
         - Divider block
       - `build_confirmed_card_blocks(card: SignalCard, action: str) -> list[dict]`:
         - Shows card title with action result (e.g. "Approved" with checkmark, "Rejected" with X, "Snoozed for 30 days" with clock)
         - No buttons (buttons removed after action)
       - `build_details_blocks(card: SignalCard, score: RubricScore) -> list[dict]`:
         - Shows all 4 dimensions: name, score, and rationale for each
         - Used as ephemeral message or appended to the card
       - `build_review_footer(showing: int, total: int) -> list[dict]`:
         - Context block: "Showing {showing} of {total} signals today" per user decision

    4. Create `src/watchman/slack/actions.py`:
       - `register_actions(app: App) -> None`: registers all action handlers on the app
       - `@app.action("approve_card")` handler: ack() immediately, parse card_id from `body["actions"][0]["value"]`, call `asyncio.run(CardRepository.set_review_state(card_id, "approved"))` via DB connection, call `client.chat_update()` with confirmed blocks (no buttons). Store `slack_message_ts` and `slack_channel_id` on the card.
       - `@app.action("reject_card")` handler: same pattern, state = "rejected"
       - `@app.action("snooze_card")` handler: ack(), call `asyncio.run(CardRepository.snooze_card(card_id, days=30))`, update message with snoozed confirmation
       - `@app.action("details_card")` handler: ack(), load card and score from DB, post ephemeral message with `build_details_blocks()` using `client.chat_postEphemeral()`
       - All handlers get `db_path` from `os.environ.get("WATCHMAN_DB_PATH", "watchman.db")` or a module-level config. Use `get_connection` context manager from storage.database.
       - Handle errors gracefully: if DB update fails, log error and post ephemeral error message

    5. Create `src/watchman/slack/commands.py`:
       - `register_commands(app: App) -> None`: registers slash command handlers
       - `@app.command("/watchman")` handler:
         - ack() immediately
         - Parse `command["text"]` — if starts with "add-source", extract URL and optional tier (default 2)
         - Validate URL (basic URL format check)
         - Validate tier is 1, 2, or 3
         - Auto-detect source type: check if URL ends in `/feed`, `/rss`, or returns RSS content type -> "rss"; else "scrape"
         - Append new source entry to `sources.yaml` (load existing, add entry, write back)
         - Respond with confirmation: "Added {url} as Tier {tier} {type} source"
         - On validation error, respond with usage instructions
         - Handle "help" subcommand: respond with available commands

    6. Create `src/watchman/slack/delivery.py`:
       - `async def deliver_daily_review(db_path: Path, rubric_path: Path) -> int`:
         - Load rubric config to get `daily_cap_target`
         - Open DB, call `CardRepository.find_top_scored_today(limit=cap_target)`
         - Also include re-queued snoozed cards (snooze_until <= now, already handled in the repository query from Plan 01)
         - Call `CardRepository.count_scored_today()` for total count
         - For each selected card, load its `score_breakdown` JSON and parse into `RubricScore`
         - Post each card to `SLACK_CHANNEL_ID` using `WebClient.chat_postMessage()` with `build_signal_card_blocks()`
         - After all cards, post the footer with `build_review_footer(showing, total)`
         - Save `slack_message_ts` and `slack_channel_id` on each card for later `chat_update`
         - Return count of cards delivered
         - Use logging for delivery progress
       - `def deliver_daily_review_sync(db_path: Path, rubric_path: Path) -> None`: sync wrapper using `asyncio.run()` for APScheduler

    7. Update `src/watchman/scheduler/jobs.py`:
       - Import `deliver_daily_review_sync` from `watchman.slack.delivery`
       - Add `schedule_delivery_job(scheduler, db_path, rubric_path)` that adds a daily job (CronTrigger at 9:00 AM) calling `deliver_daily_review_sync`
       - Export the function for use in main.py

    8. Create `tests/test_slack_blocks.py`:
       - Test `build_signal_card_blocks` returns correct block structure (section, context, actions with 4 buttons, divider)
       - Test score line format matches "X.X -- strong taxonomy fit" pattern
       - Test `build_confirmed_card_blocks` has no action blocks (buttons removed)
       - Test `build_review_footer` formats "Showing X of Y signals today"
       - Test `build_details_blocks` includes all 4 dimension scores
       - Use sample SignalCard and RubricScore fixtures
  </action>
