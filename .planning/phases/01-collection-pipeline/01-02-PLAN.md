Purpose: Collectors are the data ingestion layer. Without them, no signals enter the database. The scheduler automates collection so it runs unattended.
Output: Working collectors that fetch from all source types and a scheduler that runs them at configured intervals.
</objective>

    **src/watchman/collectors/base.py:**
    - Abstract `BaseCollector(ABC)` with:
      - `__init__(self, source: SourceConfig, db_path: Path)`: stores source config and db path
      - `async def collect(self) -> list[RawItem]`: abstract method -- fetches data from source, returns raw items
      - `async def run(self) -> int`: calls collect(), writes results to raw_items table via RawItemRepository, returns count of items written. Handles errors gracefully (log + return 0).
    - Collector factory:
      - `COLLECTOR_REGISTRY: dict[str, type[BaseCollector]] = {}`
      - `register_collector(source_type: str)` decorator
      - `get_collector(source: SourceConfig, db_path: Path) -> BaseCollector` factory function

    **src/watchman/collectors/rss.py:**
    - `@register_collector("rss") class RSSCollector(BaseCollector)`:
      - `async def collect(self) -> list[RawItem]`:
        1. Fetch feed content with httpx (timeout=30s, follow_redirects=True)
        2. Parse with feedparser.parse(content)
        3. For each entry: extract title, link, summary/description, published date
        4. Published date: try entry.published_parsed (struct_time -> datetime), fallback to entry.updated_parsed, fallback to fetch timestamp (per user decision)
        5. Create RawItem with collector_type="rss", source_name from self.source.name
        6. Store raw_data as JSON string of entry dict (for debugging/reprocessing)
        7. Log number of items found

    **src/watchman/collectors/api.py:**
    - `@register_collector("api") class APICollector(BaseCollector)`:
      - `async def collect(self) -> list[RawItem]`:
        1. Fetch JSON with httpx.AsyncClient (timeout=30s)
        2. Parse response as JSON
        3. Handle Hacker News Algolia API format specifically: endpoint `http://hn.algolia.com/api/v1/search_by_date?tags=show_hn&hitsPerPage=20` -- extract hits[].title, hits[].url, hits[].created_at, hits[].story_text
        4. For other API sources: attempt generic JSON array or object-with-results parsing
        5. Create RawItems with collector_type="api"
        6. Use published_date from API response, fallback to fetch timestamp
        7. Log items found

    **src/watchman/collectors/scrape.py:**
    - `@register_collector("scrape") class ScrapeCollector(BaseCollector)`:
      - `async def collect(self) -> list[RawItem]`:
        1. Fetch page HTML with httpx (timeout=30s, follow_redirects=True)
        2. Extract content with trafilatura.extract(html, include_links=True, output_format="txt")
        3. Extract metadata with trafilatura.extract_metadata(html)
        4. Create single RawItem per page with title from metadata.title, summary from extracted text (first 500 chars), url from source URL
        5. Published date from metadata.date if available, fallback to fetch timestamp
        6. collector_type="scrape"
        7. Log extraction result

    All collectors:
    - Use logging module (not print/console.log)
    - Include error handling with try/except for network errors, parse errors
    - Exponential backoff not needed for v1 (simple timeout + error logging sufficient)
    - Type annotations on all function signatures
