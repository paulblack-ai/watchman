    **src/watchman/processing/deduplicator.py:**
    - `async def is_duplicate(card: SignalCard, repo: CardRepository) -> tuple[bool, SignalCard | None]`:
      1. **Layer 1 -- URL hash match (fast, exact):**
         - Query CardRepository.find_by_url_hash(card.url_hash)
         - If found: return (True, existing_card)
      2. **Layer 2 -- Content fingerprint (slower, fuzzy):**
         - Get recent cards from CardRepository.find_since(cutoff = now - 7 days)
         - For each recent card: compute title similarity using difflib.SequenceMatcher
         - Threshold: >0.85 ratio (per user decision)
         - Additional guard: skip if title length < 20 chars (avoid false positives on short titles like "Update", "Release Notes" -- per research pitfall #5)
         - If match found with similar date (within 1 day): return (True, existing_card)
      3. If no match: return (False, None)

    - `async def handle_duplicate(duplicate_card: SignalCard, canonical_card: SignalCard, repo: CardRepository) -> None`:
      1. Link duplicate to canonical: repo.link_duplicate(duplicate_card.id, canonical_card.id)
      2. Increment seen_count on canonical: repo.increment_seen_count(canonical_card.id)
      3. Log: "Duplicate detected: '{duplicate_card.title}' matches canonical card {canonical_card.id}"

    Use conservative matching per user decision -- better to let some dupes through than merge distinct signals. The duplicate count ("seen across N sources") is visible on canonical card's seen_count field.

      1. Open DB connection
      2. If items_found > 0: record run via HealthRepository, reset consecutive_zeros
      3. If items_found == 0: record run, increment consecutive_zeros in the health table
      4. Check consecutive_zeros count for this source
      5. If consecutive_zeros >= 2: trigger alert (call check_and_alert)

    - `async def check_and_alert(db_path: Path, source_name: str, source_type: str, consecutive_zeros: int) -> None`:
      1. Load SLACK_BOT_TOKEN and SLACK_PAUL_USER_ID from environment (os.environ.get)
      2. If either missing: log warning "Slack credentials not configured, skipping health alert" and return
      3. If consecutive_zeros == 2: send individual alert via send_health_alert (first detection per user decision)
      4. If consecutive_zeros > 2: skip individual alert (will be in daily digest per user decision)
      5. Log alert action

    - `async def get_daily_digest(db_path: Path) -> list[dict]`:
      1. Query source_health for all sources with consecutive_zeros >= 2
      2. Return list of {source_name, consecutive_zeros, last_error} dicts
      3. Used by a daily digest job (scheduled separately)

    **src/watchman/health/alerter.py:**
    - `def send_health_alert(token: str, user_id: str, source_name: str, source_type: str, consecutive_zeros: int) -> bool`:
      1. Create slack_sdk.WebClient(token=token)
      2. Build message with source name, consecutive zero count, and suggested action:
         - rss: "Check if the feed URL has changed or the site is down"
         - api: "Check if the API key has expired or the endpoint has moved"
         - scrape: "Check if the page structure or CSS selectors have changed"
      3. Send via client.chat_postMessage(channel=user_id, text=message)
      4. Return True on success, False on error (log error, don't raise)

      3. Send via chat_postMessage
